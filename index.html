Development

Overview

In this project, I conducted a competitive analysis between three different companies to analyze the design choices made by industry experts. I looked at features related to the filtering and sorting functionality of the page. I then implemented a page of my own based on the features that I thought worked well, specifically when used in combination with a favoriting function.

Summary of Comparison

    Crunchyroll:

  • One design choice that worked well for Crunchyroll was the changing header to indicate which sort was selected. This was helpful in terms of visual hierarchy because the heading is large enough to ensure the user sees the sort change. Another factor that worked well was the grid display because the content was primarily-image based, so the grid allows the user to quickly scan the image before the text, which was minimal anyways.

  • One feature that did not work as well was the sort button, because it doesn’t indicate it’s a sort button through text, and instead uses an icon and displays the automatic sort option as the text. This was not very intuitive because it seems to have already selected a sort.

    • Github:

  • One design choice that worked well for Github was the indication of the selected filters. Although the selected filter is shown in the select button (similar to Yelp), there is also text indicating all the selected filters, which is helpful for clarity. Another design choice that worked well was the button allowing users to clear filters, which is more convenient than manually deselecting filters.

  • A design choice that did not work well was the deselect option, which doesn’t automatically remove the item from the favorites without refreshing or resorting first.

    • Yelp:

  • One design choice that worked well for Yelp was the dropdown buttons used to indicate the filter and sort categories. It made it obvious to the user that there are multiple options to choose from based on category. It also helped that the selected type remained in the select button, so the user knows the filter was applied. I also liked that Yelp condensed the layout of the favorited items into cards that only show the most important features, because it makes the design more clean and not cluttered.

  • A feature that did not work as well was the remove feature, since there was a popup that asks the user to save the item to another collection. This wouldn’t match the mental model of deselecting with the intent to remove rather than to save to another collection.

    • All Websites:

  • I found that all websites used icons to represent the favoriting function, which looked simple but still felt effective in signaling the favorites feature. This is most likely due to convention (since it is convention for hearts and stars to mean “liking” something).

  • All the filtering and sorting functions were in one line and either left aligned or right aligned, which made it more efficient to use due to grouping. All websites used filter and sort in place, so the layout stayed the same as before, just with different ordering and items. This was helpful for memorability purposes.

Implementation Plan

Considering the features that worked well for each website, I chose to include the following:

  • A dropdown sort button that is labeled “Sort By”

  • Dropdown filter buttons for each filter type (on the same line as the sort button)

  • A button to clear all filters at once

  • Icon to represent the favorite button, which can add and remove from the favorite list in one click

  • Grid layout to display all items, including the favorites

  • A header that changes according to the sort selected, and text indicating which filters are selected

  • Different layout between a favorited item and a list item. Less information in the favorited item.

Implementation

My App

The intention of my app is a movie favoriting app that allows users to keep track of their favorite movies. The original list of movies is displayed as a grid of cards, with each movie card showing an image related to the movie, as well as its title, date of release, rating, duration, and genre tags. The favorite list is in the top section, and is also displayed as a grid of cards, but the cards are smaller and do not show the genre tags. These are the three main functions of the page:

  • The filter is based on the genre and rating of the movie, (which are more typical factors that users may use to determine whether or not to watch a movie). These can be selected by using the respective dropdown buttons, and removed by selecting the option "None." They can also be cleared at once by the reset button.

  • The sort function sorts the movies alphabetically based on the title, or by the date of release from oldest to newest. The sort can be selected by using the "Sort by" dropdown button, and deselected by selecting the option "None." The sort can also be cleared by the reset button.

  • The aggregation feature is a favoriting function that lists the favorited movies, the total number of favorites, and the total duration of all movies combined (which may be helpful for scheduling when / how long to watch). To favorite a movie, select the heart icon, and to remove a movie, deselect the heart icon (either from the movie card, or from the favorite card).

Additional Design Choices

  • Favorite List: I chose to display the favorites on the top because as the user browses movies and scrolls down, they can use the scroll to top button to immediately get to the favorites.

  • Favorite Card: I chose to display the favorite card differently from the movie card by using a horizontal layout because I wanted a clear visual distinction between the two types. Plus, the favorite card has less information to display.

  • Genre Tags: On each movie card, the genre tag is clickable and will automatically filter for all movies that fall under that genre. This is to help the user in filtering (especially if they don't want to scroll all the way up).

Takeaways

After working on this project, I can see how important it is to do a copmarison between different websites. I found it helpful that I could take inspiration from other designs and reuse some feature ideas for my own page. As for the implementation process, I thought it was challenging because I struggled to keep components consistently updated according to the state (like the heart icon, for example). However, after working through those issues, I feel like a bit more comfortable using React and MaterialUI. If I were to build on this project, I might add even more filtering functions, like a search by text feature.
